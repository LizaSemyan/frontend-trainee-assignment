# Тестовое задание для стажёра Frontend (осенняя волна 2025)

## Описание проекта

**Система управления объявлениями для модерации**

![scheme](<https://github.com/avito-tech/tech-internship/blob/main/Tech%20Internships/Frontend/Frontend-trainee-assignment-autumn-2025/schema%20(3).jpg>)

Веб-приложение для модерации объявлений на платформе Авито. Это упрощённая версия внутренней системы, которую используют модераторы для проверки и управления объявлениями пользователей.

### **Бизнес-контекст**

Ежедневно на Авито публикуются тысячи объявлений, которые должны пройти модерацию перед публикацией. Модераторам нужен удобный инструмент для быстрой проверки объявлений, принятия решений и отслеживания своей эффективности.

#### **1. Главная страница — Список объявлений (/list)**

- Отображение списка объявлений в виде карточек
- Каждая карточка содержит:
  - Изображение товара (можно использовать placeholder-изображения)
  - Название объявления
  - Цена
  - Категория
  - Дата создания
  - Статус (на модерации / одобрено / отклонено)
  - Индикатор приоритета (обычный / срочный)

**Фильтрация и поиск:**

- Фильтр по статусу (множественный выбор)
- Фильтр по категории
- Фильтр по диапазону цен
- Поиск по названию объявления
- Сброс всех фильтров

**Сортировка:**

- По дате создания (новые/старые)
- По цене (возрастание/убывание)
- По приоритету

**Пагинация:**

- По 10 объявлений на страницу
- Навигация между страницами
- Отображение общего количества объявлений

#### **2. Страница детального просмотра объявления (/item/:id)**

При клике на карточку открывается детальная страница объявления.

**Информация об объявлении:**

- Галерея изображений (минимум 3 изображения)
- Полное описание
- Характеристики товара (в виде таблицы ключ-значение)
- Информация о продавце:
  - Имя
  - Рейтинг
  - Количество объявлений
  - Дата регистрации

**История модерации:**

- Список всех действий с объявлением
- Кто проверял (имя модератора)
- Когда (дата и время)
- Какое решение принял
- Комментарий (если был)

**Панель действий модератора:**

- Кнопка «Одобрить» (зелёная)
- Кнопка «Отклонить» (красная)
- Кнопка «Вернуть на доработку» (жёлтая)

**При отклонении:**

- Обязательное поле для указания причины
- Быстрые шаблоны причин:
  - Запрещённый товар
  - Неверная категория
  - Некорректное описание
  - Проблемы с фото
  - Подозрение на мошенничество
  - Другое (с полем ввода)

**Навигация:**

- Кнопка «Назад к списку»
- Кнопки «Предыдущее» / «Следующее» объявление (для быстрой модерации)

#### **3. Страница статистики модератора (/stats)**

**Общая статистика:**

- Карточки с метриками:
  - Всего проверено объявлений (за сегодня/неделю/месяц)
  - Процент одобренных
  - Процент отклоненных
  - Среднее время на проверку одного объявления
- Графики:
  - График активности по дням за последнюю неделю (столбчатая диаграмма)
  - Круговая диаграмма распределения решений (одобрено/отклонено/на доработку)
  - График по категориям проверенных объявлений

## Запуск проекта

После клонирования репозитория выполните команду в корне проекта:

```bash
docker compose up --build
```

После запуска

- Клиент доступен по http://localhost:5173.
- Бэкенд доступен по http://localhost:3001.

## Стек технологий

- Node.js v20.
- React v19.
- TypeScript. Использование TypeScript повышает надёжность приложения благодаря статической типизации, благодаря чему повышает предсказуемость кода и способствует раннему выявлению ошибок. В приложении работа идет с одними и теми же сущностями, соответсвенно типизация помогает избежать возможных ошибок.
- react-router-dom для роутинга.
- Vite. Простая сборка проекта без долгой ручной настройки сборщика, моментальные перезапуски во время разработки.
- Redux (cтейт-менеджмент). Использован для хранения состояния списка объявлений для реализации перехода между объявлениями на странице модерации объявления.
- Axios. Удобный инструмент для работы с API.
- Material UI (MUI). Предоставляет готовые компоненты, что позволило ускорить разработку.
- Recharts для визуализации графиков. Прост в использовании, умеет строить требуемые в задании диаграммы.

## Архитектура

Структура проекта:

client/

- public/
  -- Статические файлы, попадающие в итоговую сборку без изменений.

- src/

  - api/
    -- axios-инстанс (единая точка входа для запросов)
    -- сервисы API

  - assets/
    -- иконки

  - components/
    -- Модуль переиспользуемых компонентов.

  - constants/
    -- Статичные значения (лэйблы и тд)

  - hooks/
    -- Логические хуки. Выделение бизнес-логики вне компонентов

  - pages/
    -- /list — список объявлений
    -- /item/:id — модерация конкретного объявления
    -- /stats — метрики + графики
    -- Страница для неизвестных маршрутов

  - router/
    -- Основной роутинг приложения (React Router)

  - store/
    -- настройка Redux Toolkit

  - types/
    -- Полная типизация сущностей с backend API
    -- Типы DTO, интерфейсы, ответы запросов

  - utils/
    -- Вспомогательные функции:
    --- форматирование дат
    --- преобразование данных для графиков
    --- работа с поиском/фильтрами

  - App.tsx
    -- Корневой компонент

  - main.tsx
    -- Точка входа в App

  - theme.ts
    -- MUI Theme Provider

## Особенности реализации

- Использование готового API, расположенного в папке tech-int3-server этого репозитория.
- Возможность запуска проекта в контейнеризированной среде с помощью Docker.  
  Клиент и сервер поднимаются одной командой через `docker compose up --build`.
- Настроен Prettier для автоформатирования кода. Позволяет не тратить время на приведение кода в единообразный вид, экономя время.
- Настроен ESLint для отслеживания потенциальных ошибок и их своевременного устранения.

Из дополнительных функций реализовано:

- Реализован фильтр по периоду на странице статистики: сегодня, 7 дней, 30 дней, произвольный период.
